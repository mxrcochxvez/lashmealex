# Repo Rules

- Use atomic commit scopes and Karma-style commit messages to commit code piecemeal. This ensures that each change is as small and clean as possible, allowing for easier reverts of problematic changes with minimal blast radius.

- Prefer separate commits for workspace setup, backend scaffolding, storefront UI, and follow-up fixes instead of bundling unrelated changes together.
- Do NOT use the chrome browser or any UI inspection tools unless explicitly requested by the user.
